1. The Core Problem
Nigerian farmers lose 30–40% of yield to timing errors — watering too late, pest detection too late, planting at the wrong time. The data to prevent this already exists. The gap is translation: converting satellite and weather data into simple, actionable advice delivered through channels farmers already use.

2. Solution Architecture
The platform operates in three layers: data ingestion from free satellite and weather APIs, a processing engine that runs the agronomic models, and a delivery layer that converts results into plain-language WhatsApp and SMS messages.

DATA SOURCES (Free / Open)
NASA SMAP (Soil Moisture) · GFS Weather · Open-Meteo API · Ogimet Stations · Pest Degree-Day Database

PROCESSING ENGINE (Python / FastAPI)
Spatial downscaling · Penman-Monteith ET calculation · Pest degree-day models · Soil moisture trend projection · Translation layer (data → advice)

DELIVERY LAYER
WhatsApp Cloud API (Meta) — 1,000 free conversations/month
SMS Gateway — Africa's Talking (~₦4–₦8/SMS, pay-per-use)
Future: Mobile App

Architecture Flow
Every 6 hours: satellite soil moisture and weather forecast data is fetched for all registered farm locations. The processing engine runs ET calculations, degree-day pest accumulations, and soil moisture projections. Results pass through the translation engine. Alerts are queued and sent at 6 AM local time daily, or immediately for high-risk pest warnings.

3. The Three Core Services
Service 1 — Soil Moisture Forecast (Without Physical Sensors)
How It Works
Pull soil moisture from NASA SMAP (9km resolution, 3-day satellite revisit) or ESA CCI (combined satellite product)
Pull 7-day weather forecast from Open-Meteo API (free, global, no API key needed)
Calculate evapotranspiration (ET) using the Penman-Monteith equation driven by forecast temperature, humidity, wind speed, and solar radiation
Soil moisture trend = current satellite estimate minus projected ET loss over next 3–5 days
When trend crosses the crop-specific threshold → trigger alert to farmer

How It Works

For Nigeria: NiMet station data or Ogimet for bias correction where available
Elevation-based downscaling using SRTM data: adjust temperature by lapse rate (6.5°C per 1000m elevation gain)
Convert raw meteorology into farmer-relevant actions: not 'humidity 65%' but 'Good day for spraying' or 'Rain likely tomorrow — delay fertilizer application'

Layer 2 — Processing Engine
Step 1: Location Resolution
'Sabon Gari, Kaduna' → latitude/longitude via a geocoding database pre-loaded with Nigerian LGAs and towns. This runs once at farmer registration and is stored in the database.
Step 2: ET Calculation (Penman-Monteith)
The reference evapotranspiration (ET₀) tells us how much water the atmosphere is demanding from the soil each day. This drives the soil moisture forecast.
ET₀ (mm/day) = [0.408 × Δ × (Rn − G) + γ × (900 / (T + 273)) × u₂ × (es − ea)]
               ─────────────────────────────────────────────────────────────────────
               [Δ + γ × (1 + 0.34 × u₂)]

Where:
  Δ   = slope of saturation vapour pressure curve (kPa/°C)
  Rn  = net radiation (MJ/m²/day)
  G   = soil heat flux (≈ 0 for daily calculations)
  γ   = psychrometric constant (kPa/°C)
  T   = mean daily temperature (°C)
  u₂  = wind speed at 2m height (m/s)
  es  = saturation vapour pressure (kPa)
  ea  = actual vapour pressure (kPa)
Step 3: Soil Moisture Projection
SM_day3 = SM_current − Σ(ET₀ − Rainfall) for days 1, 2, 3

If SM_day3 < crop_threshold → trigger irrigation alert
If SM_day3 still adequate → no alert, check again in 24h
Step 4: Degree-Day Accumulation (Pest Model)
DD_daily = max(0,  (Tmax + Tmin) / 2  −  Tbase)

DD_accumulated = sum of all DD_daily since season start

Progress (%) = (DD_accumulated / DD_threshold) × 100

Thresholds:
  > 80% → HIGH RISK — alert farmer immediately
  > 60% → MEDIUM   — advisory warning
  > 40% → LOW      — monitor conditions
Step 5: Translation Engine (Data → Plain Language)
Every numeric output is converted to an action-oriented message in plain English. The rule set is crop-specific. No raw numbers are sent to the farmer — only status labels and clear actions.

6. Penman-Monteith — Full Python Implementation
This is the agronomic core of FarmSmart. Every soil moisture forecast depends on this calculation being accurate. The function takes inputs directly from the Open-Meteo API response.

import math

def calculate_eto(
    temperature_c: float,       # Mean daily temp (°C) from Open-Meteo
    humidity_percent: float,     # Relative humidity (%)
    wind_speed_ms: float,        # Wind speed at 2m height (m/s)
    solar_radiation_mj: float,   # Solar radiation (MJ/m²/day)
    elevation_m: float,          # Farm elevation (from SRTM)
    latitude_deg: float,         # Farm latitude
    day_of_year: int             # Julian day (1–365)
) -> float:
    """
    Returns ET₀ in mm/day — atmospheric water demand
    """
    # 1. Atmospheric pressure (kPa)
    P = 101.3 * ((293 - 0.0065 * elevation_m) / 293) ** 5.26

    # 2. Psychrometric constant (kPa/°C)
    gamma = 0.000665 * P

    # 3. Saturation vapour pressure (kPa)
    es = 0.6108 * math.exp((17.27 * temperature_c) / (temperature_c + 237.3))

    # 4. Actual vapour pressure (kPa)
    ea = es * (humidity_percent / 100)

    # 5. Vapour pressure deficit
    vpd = es - ea

    # 6. Slope of saturation vapour pressure curve (kPa/°C)
    delta = (4098 * es) / ((temperature_c + 237.3) ** 2)

    # 7. Net radiation
    Ra = _extraterrestrial_radiation(latitude_deg, day_of_year)
    Rns = 0.77 * solar_radiation_mj
    Rnl = 4.903e-9 * ((temperature_c + 273.16) ** 4) * (0.34 - 0.14 * math.sqrt(ea))
    Rn = Rns - Rnl

    # 8. Soil heat flux (negligible for daily calc)
    G = 0

    # 9. Penman-Monteith equation
    numerator = (0.408 * delta * (Rn - G)) + \
                (gamma * (900 / (temperature_c + 273)) * wind_speed_ms * vpd)
    denominator = delta + (gamma * (1 + 0.34 * wind_speed_ms))

    return max(0, numerator / denominator)   # mm/day

def _extraterrestrial_radiation(lat_deg: float, doy: int) -> float:
    lat_rad = math.radians(lat_deg)
    delta   = 0.409 * math.sin((2 * math.pi / 365) * doy - 1.39)
    omega_s = math.acos(-math.tan(lat_rad) * math.tan(delta))
    d_r     = 1 + 0.033 * math.cos((2 * math.pi / 365) * doy)
    G_sc    = 0.0820
    Ra = (24 * 60 / math.pi) * G_sc * d_r * (
         omega_s * math.sin(lat_rad) * math.sin(delta) +
         math.cos(lat_rad) * math.cos(delta) * math.sin(omega_s))
    return Ra

Practical interpretation:
If ET₀ = 5 mm/day and rainfall = 0, the soil loses 5mm of water that day.
Over 3 days with no rain = 15mm cumulative loss.
If soil moisture drops below the crop's critical threshold → irrigation alert fires.

7. Pest Degree-Day Model — Full Python Implementation

class PestModel:
    def __init__(self, name: str, base_temp_c: float, dd_threshold: float):
        self.name          = name
        self.base_temp     = base_temp_c
        self.dd_threshold  = dd_threshold
        self.accumulated   = 0.0
        self.generations   = 0

    def add_daily_reading(self, tmax: float, tmin: float) -> dict:
        tmean    = (tmax + tmin) / 2
        daily_dd = max(0.0, tmean - self.base_temp)
        self.accumulated += daily_dd

        # Count completed generations
        while self.accumulated >= self.dd_threshold:
            self.accumulated -= self.dd_threshold
            self.generations += 1

        progress = (self.accumulated / self.dd_threshold) * 100

        if progress > 80:
            risk = 'HIGH'
        elif progress > 60:
            risk = 'MEDIUM'
        elif progress > 40:
            risk = 'LOW'
        else:
            risk = 'MINIMAL'

        return {
            'pest':                self.name,
            'daily_dd':            round(daily_dd, 1),
            'accumulated_dd':      round(self.accumulated, 1),
            'threshold':           self.dd_threshold,
            'progress_percent':    round(progress, 1),
            'generations':         self.generations,
            'risk_level':          risk,
        }

# Registered pest models (Nigerian crops)

    'desert_locust': PestModel('Desert Locust',     base_temp_c=15.0, dd_threshold=400),
    'stem_borer':    PestModel('Maize Stem Borer',  base_temp_c=10.0, dd_threshold=250),

def check_humidity_pest_risk(humidity_percent: float, leaf_wetness_hours: int) -> str:
    """Humidity-triggered disease risk (Late Blight, Rust)"""
    if humidity_percent > 85 and leaf_wetness_hours > 48:
        return 'HIGH'
    elif humidity_percent > 70 and leaf_wetness_hours > 24:
        return 'MEDIUM'
    return 'LOW'

8. WhatsApp Bot — FastAPI Implementation
The webhook server receives messages from Meta's WhatsApp Cloud API and routes them to the correct handler. The server also handles the Meta webhook verification handshake at startup.

from fastapi import FastAPI, Request
import requests, json, os

app = FastAPI()

WHATSAPP_TOKEN  = os.environ['WHATSAPP_TOKEN']
PHONE_NUMBER_ID = os.environ['PHONE_NUMBER_ID']
VERIFY_TOKEN    = os.environ['VERIFY_TOKEN']

@app.get('/webhook')
async def verify_webhook(request: Request):
    """Meta verifies this endpoint at WhatsApp Cloud API setup"""
    mode      = request.query_params.get('hub.mode')
    token     = request.query_params.get('hub.verify_token')
    challenge = request.query_params.get('hub.challenge')
    if mode == 'subscribe' and token == VERIFY_TOKEN:
        return int(challenge)
    return {'error': 'Verification failed'}, 403

@app.post('/webhook')
async def receive_message(request: Request):
    body     = await request.json()
    entry    = body.get('entry', [{}])[0]
    changes  = entry.get('changes', [{}])[0]
    value    = changes.get('value', {})
    messages = value.get('messages', [])

    if not messages:
        return {'status': 'no_message'}

    message = messages[0]
    phone   = message['from']
    text    = message.get('text', {}).get('body', '').strip().upper()

    farmer   = get_farmer(phone)
    response = route_command(phone, farmer, text)

    if response:
        send_whatsapp_message(phone, response)

    return {'status': 'processed'}

9. Translation Engine — Data to Plain Language
This is the most important layer of the system. Raw numbers and probabilities confuse farmers. Every data point is converted to a status label and a single clear action. No farmer should ever receive a number they cannot act on.

def translate_soil_moisture(
    sm_value: float,    # Current soil moisture (m³/m³) from NASA SMAP
    sm_trend: float,    # Projected 3-day change
    crop: str           # Registered crop
) -> str:

    THRESHOLDS = {
        'maize':   {'critical': 0.15, 'low': 0.20},
        'cassava': {'critical': 0.10, 'low': 0.15},
        'tomato':  {'critical': 0.18, 'low': 0.23},
        'rice':    {'critical': 0.30, 'low': 0.35},
        'yam':     {'critical': 0.12, 'low': 0.18},
    }
    t = THRESHOLDS.get(crop, THRESHOLDS['maize'])

    if sm_value < t['critical']:
        status = '🔴 CRITICALLY LOW'
        action = 'Irrigate IMMEDIATELY'
    elif sm_value < t['low']:
        status = '🟡 LOW'
        action = 'Irrigate within 24 hours'
    else:
        status = '🟢 ADEQUATE'
        action = 'No irrigation needed now'

    if sm_trend < -0.02:
        trend = '📉 Dropping fast — soil drying quickly'
    elif sm_trend < -0.01:
        trend = '📉 Dropping slowly'
    elif sm_trend < 0:
        trend = '→ Stable'
    else:
        trend = '📈 Improving (recent rain helped)'

    return f'''
🌱 *Soil Moisture Status*
Crop: {crop.title()}

Status: {status}
{trend}

💧 *Recommended Action:*
{action}

_Check again tomorrow or reply SOIL_
'''.strip()

def translate_pest_risk(dd_data: dict, humidity_risk: str) -> str | None:
    is_high = dd_data['risk_level'] == 'HIGH' or humidity_risk == 'HIGH'
    is_med  = dd_data['risk_level'] == 'MEDIUM' or humidity_risk == 'MEDIUM'

    if is_high:
        return f'''
⚠️ *PEST ALERT — {dd_data['pest']}*
Risk Level: 🔴 HIGH

Your crop is in the danger window.

🛡 What to do:
1. Scout your farm tomorrow morning
2. Look for early signs of damage
3. Consider preventive spray
4. Contact your extension agent

Early action prevents 70% of crop loss!
_Reply SIGNS to learn what to look for_
'''.strip()

    if is_med:
        return f'''
⚠️ *Pest Advisory — {dd_data['pest']}*
Risk Level: 🟡 MODERATE

Conditions building. No emergency yet.
• Monitor your farm twice this week
• Check undersides of leaves
• Prepare your spray
_No immediate action required_
'''.strip()

    return None   # No alert needed
