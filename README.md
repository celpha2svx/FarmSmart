## What it does (for the farmer)

FarmSmart is a free Android app that turns a phone into a farming assistant. It tells the farmer:

- **What to do today** — "Apply urea to your maize tomorrow morning. Soil moisture is right."
- **What's coming** — "Heavy rain in 48 hours. Postpone fertilizer."
- **What's attacking the field** — Snap a photo of the leaf, get a real diagnosis with treatment and prevention.
- **What the market is paying** — Live AFEX prices for maize, rice, beans, millet, etc., with a 7-day trend chart.
- **When the season is shifting** — Rainy season starts in 12 days → planting window opens.

In **English, Hausa, Yoruba, or Igbo** from the very first screen. Works on any Android 8+ phone. ~93 MB universal APK; ~34 MB for arm64 devices.

## What you can do today (after Phase 1)

1. Open the app and **pick your language** — first thing, no signup walls.
2. **Sign up with your phone** — OTP via Africa's Talking (SMS) or voice.
3. **Set a 4-digit PIN** so next time you sign in without SMS — even on poor signal.
4. **Tell us about your farm** — pick the crops you grow, type your town or LGA, pick farm size, give a planting date per crop.
5. **Get your first advisory** — "Good morning, Aisha. Your maize is 32 days old. Apply NPK 15-15-15 this week. Watch for fall armyworm — pressure is high in your area."
6. **Scan a leaf** — get a real diagnosis (or an honest "we can't tell, try again").
7. **Plan the week** — calendar of planting, fertilizing, scouting, harvest tasks.
8. **Check market prices** — see what maize sells for in Kano, Lagos, Ibadan today vs last week.
9. **Send feedback** — actually reaches a human via GitHub Issues.

The advisory is generated by a rules engine, not a chatbot. It draws on your crop, your planting date, your local weather (from Open-Meteo), and your soil (from FAO WaPOR). It gives you **verbs**: *apply, scout, water, harvest* — not paragraphs.

## What's *not* done yet (be honest)

- **Drift database** is declared but not used. The app reads from the backend, not from a local cache. Offline mode is a Phase 5 task.
- **FCM push notifications** aren't wired. The `firebase/` folder is empty. Local task reminders are stubbed for Phase 6.
- **PIN change / account recovery without OTP** isn't built. Forgot your PIN → sign in with OTP → set a new PIN.
- **The full Hausa / Yoruba / Igbo advisory** is in progress. Phase 1 ships the UI strings in all three languages; the advisory prose itself is currently English. Translating the rules engine output is a content task we need a real Yoruba/Hausa/Igbo speaker for.
- **HuggingFace pest detection** depends on you having an API token configured. Without it, the scanner honestly says "we can't identify this" instead of inventing a result.

## Tech stack

**Frontend** — Flutter 3.29 · Riverpod · Dio · Drift (schema declared, wired in Phase 5) · fl_chart · table_calendar · image_picker · pinput · flutter_secure_storage · flutter_local_notifications (Phase 6)

**Backend** — Python 3.12 · FastAPI · Uvicorn · SQLAlchemy · APScheduler · httpx · Africa's Talking (SMS) · HuggingFace Inference API (pest detection) · Open-Meteo (weather) · FAO WaPOR (NDVI) · AFEX Exchange (market prices)

**CI/CD** — GitHub Actions (`build-apk.yml` builds a release APK on tag, `deploy.yml` runs pytest)

## Setup

### Backend (local)

```bash
python -m venv .venv
.venv/Scripts/activate              # Windows
source .venv/bin/activate           # Mac/Linux
pip install -r requirements.txt
cp .env.example .env                # fill in HUGGINGFACE_TOKEN etc.
uvicorn app_api:app --reload --port 8000
```

### Mobile app (local)

```bash
cd farmsmart_app
flutter pub get
dart run build_runner build --delete-conflicting-outputs
flutter run                          # debug on connected device
```

### Environment variables

```
DATABASE_URL=postgresql://...           # SQLite by default for dev
APP_ENV=development                     # 'development' returns OTP in API response
HUGGINGFACE_TOKEN=hf_...                # enables real pest detection
HUGGINGFACE_MODEL=pierreguillou/nlp-v2.0-classifier-plant-disease
AT_API_KEY=...                          # Africa's Talking — for OTP SMS in production
AT_USERNAME=sandbox
FEEDBACK_WEBHOOK_URL=https://...        # Cloudflare Worker that creates GitHub Issues
```

## API

See [API_CONTRACT.md](API_CONTRACT.md) for the full contract (paths, request/response shapes, auth). Every endpoint returns the same envelope:

```json
{ "status": "ok" | "error", "data": <payload> | null, "error": { "code", "message" } | null }
```

Highlights:

- `POST /api/auth/send-otp` — request OTP (returns `dev_code` in dev mode)
- `POST /api/auth/verify-otp` — exchange code for token
- `POST /api/auth/set-pin` · `POST /api/auth/login-pin` — passwordless return login
- `POST /api/farm/register` — multi-crop + per-crop planting date
- `POST /api/advisory/generate` — weather + soil + actionable advice
- `GET  /api/tasks?phone=…&crop=…&date=YYYY-MM-DD` — per-day task list
- `POST /api/pest/detect` — multipart image upload (returns honest "unable to identify" on failure)
- `GET  /api/market/prices?crop=maize` — AFEX prices + 7-day series
- `GET  /api/announcements` — admin broadcasts
- `GET  /api/locations/search?q=iba` — type-ahead for LGAs
